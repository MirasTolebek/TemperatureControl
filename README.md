# 🏠 Автономный контроллер циркуляционного насоса отопления на Arduino
Максимально надежный контроллер управления циркуляционным насосом для домашней печи с водяным контуром или твердотопливного котла. Проект изначально планировался на базе ESP32 с Wi-Fi, но для повышения стабильности и отказоустойчивости было принято решение перейти на «железобетонную» логику на базовом микроконтроллере Arduino без использования интернета.
Никаких лишних дисплеев или зависающего Wi-Fi. Настройка, управление гистерезисом и диагностика осуществляются локально с любого Android-смартфона через USB OTG прямо в котельной.
## ✨ Особенности проекта
* **Абсолютная автономность:** Задал параметры включения/выключения один раз — работает годами.
* **Сохранение настроек:** Целевая температура и гистерезис сохраняются в энергонезависимую память (EEPROM) с алгоритмом защиты от износа (`EEPROM.update()`).
* **Failsafe-логика (Защита от закипания котла):** Критически важная функция для твердотопливных котлов! Если термодатчик отвалится, перетрется провод или он выйдет из строя, Ардуино автоматически запустит циркуляционный насос на постоянную работу для предотвращения закипания воды и аварии в системе.
* **Режим тишины (Умный логгинг):** Плата не перегружает порт данными по умолчанию. Вывод температурного лога можно включить/выключить со смартфона.
* **Экономия SRAM:** Весь текстовый интерфейс обернут в макрос `F()`, что сохраняет оперативную память микроконтроллера свободной для стабильной работы 24/7.
## 🛠 Оборудование и схема подключения
* **Arduino** (Nano, UNO или Pro Mini).
* **Датчик температуры:** Водонепроницаемый `DS18B20` (в металлической гильзе, крепится на трубу подачи воды) с модулем Terminal Adapter (со встроенным подтягивающим резистором).
* **Модуль реле:** Стандартное реле Arduino с инверсной логикой (включение по низкому уровню `LOW`) для управления питанием циркуляционного насоса 220В.
### Таблица пинов (Wiring)

| Компонент | Пин компонента | Пин Arduino |
| :--- | :--- | :--- |
| **DS18B20** | VCC | 5V |
|  | GND | GND |
|  | DAT | D2 |

## 📱 Настройка контроллера через смартфон
Настройка параметров производится прямо в котельной со смартфона:
1. Понадобится Android-смартфон, кабель **USB OTG** и приложение **Serial USB Terminal** из Google Play.
2. Подключите Ардуино к телефону через OTG-переходник.
3. В настройках терминала укажите **Baud rate: 9600**.
4. При подключении плата выдаст приветствие со списком команд и текущими уставками.
### 📜 Список команд (отправлять в Serial)

| Команда | Описание действия |
| :--- | :--- |
| `S` или `STATUS` | Запросить текущую температуру трубы и статус насоса (1 раз). |
| `M1` | Включить постоянный мониторинг (бегущий лог температуры раз в секунду). |
| `M0` | Выключить мониторинг (перевести плату в режим тишины). |
| `T65` | Задать температуру **включения** насоса на 65°C (диапазон 10-100). |
| `H10` | Задать **гистерезис** выключения на 10°C. Насос выключится при падении температуры воды до 55°C (65-10). |

## ⚠️ Рекомендации по прошивке с Android (через ArduinoDroid)
Если вы хотите прошивать плату прямо со смартфона через приложение **ArduinoDroid**, обратите внимание на следующие настройки:
1. **Настройки платы:** В меню `Settings -> Board` убедитесь, что выбран правильный тип платы. Для Arduino Nano выберите `Nano w/ ATmega328`.
2. **Загрузчик (Uploader):** В настройках приложения (`Settings -> App settings -> Uploader`) выберите режим: `Встроенный, если поддерживает тип платы. Иначе Avrdude`.
3. **Кабель:** Используйте классический переходник **USB OTG -> обычный USB-A кабель от Arduino**. Кабели `Type-C на Type-C` использовать не рекомендуется, так как ОС Android может блокировать прямой доступ программатора к последовательному порту. 
4. **Сброс (Reset):** Если возникает ошибка `stk500_getsync()`, кратковременно нажмите кнопку `RESET` на плате Arduino в момент начала загрузки скетча.
